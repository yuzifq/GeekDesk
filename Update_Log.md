### 缩放变化后贴边隐藏自动重置

问题：

主面板进入贴边隐藏状态后，如果运行期间调整 Windows 显示缩放，旧隐藏坐标仍然属于调整前的 DPI 坐标系。后续鼠标靠近边缘尝试唤出时，隐藏动画可能从旧坐标飞向新屏幕边界，出现大幅滑动、跳动或闪一下又收回的异常动画。

修复：

- 贴边隐藏状态不再保存固定 `Left` / `Top` 坐标，也不再每 1 秒写回隐藏坐标。
- 取消上一版隐藏边缘锚点方案，避免缩放变化后仍由旧锚点参与唤出动画。
- 调整显示缩放、分辨率或屏幕边界后，如果主面板正处于贴边隐藏状态，会先按上一次隐藏方向恢复到当前屏幕可见边缘。
- 自动恢复时使用底层、不激活的方式显示主面板，尽量不打断用户当前正在操作的窗口。
- 主面板恢复到边缘后继续交给原贴边隐藏计时器处理，鼠标不在面板上时会再次正常收回。
- 用户主动呼出主面板时重新按设置中的“置于顶层”选项恢复窗口层级，避免缩放变化后的底层状态影响正常显示。
- 保留隐藏方向兜底判断，隐藏方向记录丢失时按当前隐藏坐标离哪条边最近来恢复。
- 未捕获 UI 异常日志继续记录真实异常对象和调用栈，方便后续定位低概率问题。

相关文件：

- `Util/LogUtil.cs`

### 便携目录首次启动修复

问题：

打包后解压出来的便携目录如果存在 0 字节 `Data` 文件，程序启动时会把它当作已有数据文件读取，反序列化失败后进入备份恢复流程；当便携目录没有 `bak` 目录时，恢复流程会再次异常，导致进程存在但主窗口没有正常显示。

修复：

- `Data` 文件不存在或长度为 0 时，按首次运行处理，直接生成默认数据文件。
- 数据恢复流程进入备份目录前先确保 `bak` 目录存在，避免恢复逻辑二次异常。
- 保存菜单密码备份前自动创建 `bak` 目录，提升新解压目录和便携目录的容错性。

相关文件：

### 右侧贴边唤出后仍停在屏幕外修复

问题：

右侧贴边隐藏后，鼠标靠近右侧边缘时窗口已经被唤醒为可见状态，但窗口坐标仍停留在屏幕外隐藏位置，导致看起来像“不会出来”。

原因：

右侧隐藏态的触发条判断和恢复动画判断使用了两套边缘匹配逻辑。高 DPI 下 `window.Left` 和 `screenRight - showMarginWidth` 会出现小数级误差，触发条可以命中，但后续右侧恢复分支因为 `0.1` 的过小容差没有匹配，于是只执行了 `window.Show()`，没有执行 `RIGHT_SHOW` 位置恢复动画。

修复：

- 新增统一的隐藏边缘识别方法，先判断当前处于左侧、上方还是右侧隐藏态。
- 触发条判断和恢复动画复用同一个隐藏边缘结果，避免一边命中、一边不命中。
- 右侧隐藏位置使用统一容差，兼容 125% / 150% 缩放下的小数坐标误差。
- 关闭贴边隐藏时的恢复逻辑也同步复用该隐藏边缘识别。

相关文件：

- `Util/MarginHide.cs`

验证：

- 已在当前 Windows 桌面复现原问题：右侧隐藏后鼠标移到边缘，窗口会变为可见但仍停在 `2047..2877` 的屏幕外坐标。
- 修复后重新覆盖运行目录并复测：右侧隐藏后鼠标移到边缘，窗口从隐藏坐标恢复到主面板可见位置。
- Debug 构建通过，0 个错误。
- Release 构建通过，0 个错误。

### 右侧贴边高 DPI 唤出修复

问题：

Windows 显示缩放为 100% 时，主窗口靠右侧屏幕贴边隐藏后，鼠标靠近右侧边缘可以正常唤出；切换到 125% 或 150% 缩放后，同样靠近右侧边缘不会唤出。

原因：

之前只统一了“鼠标是否在窗口内”的 DPI 坐标判断，但右侧贴边隐藏/唤出仍混用了 Win32 物理屏幕坐标、WPF 逻辑窗口坐标，以及折叠窗口状态下可能失效的 `PresentationSource`。高 DPI 下隐藏条位置和鼠标热区错位，导致鼠标靠近右边缘时无法命中唤出条件。

修复：

- 新增屏幕物理坐标到 WPF 逻辑坐标的统一转换方法。
- 使用 Win32 虚拟屏幕物理边界计算屏幕范围，再按当前窗口 DPI 转换成 WPF 逻辑边界。
- 隐藏态唤出不再依赖“鼠标是否在完整窗口内”，改为单独判断鼠标是否进入屏幕边缘触发条。
- `PresentationSource` 不可用时，改用窗口句柄 `HwndSource` 获取 DPI 矩阵，最后再退回到 `VisualTreeHelper.GetDpi`。

相关文件：

- `Util/MouseUtil.cs`
- `Util/MarginHide.cs`

验证：

- Release 构建通过，0 个错误。

### 贴边隐藏误触发修复

问题：

开启贴边隐藏时，鼠标移动到窗口右侧部分区域会被误判为已经离开窗口，导致面板提前隐藏。该问题在 Windows 显示缩放不是 100% 时更容易出现。

原因：

贴边隐藏逻辑使用 Win32 `GetCursorPos` 得到屏幕物理像素坐标，但窗口位置和尺寸使用的是 WPF 逻辑坐标。两套坐标系直接比较后，窗口右侧区域会被提前判定为窗口外。

修复：

- 新增统一的鼠标窗口内坐标判断。
- 用 WPF DPI 转换把屏幕物理像素转换到窗口逻辑坐标。
- 贴边隐藏和通用 `MouseInWindow` 判断都复用同一套逻辑。
- 修正右侧屏幕边界计算，降低多显示器场景下的误判风险。

相关文件：

- `Util/MouseUtil.cs`
- `Util/MarginHide.cs`

### 主界面呼出和拖动流畅度优化

问题：

主窗口从屏幕中间呼出时，原有横向展开动画显得生硬，并且在透明窗口、大面积模糊背景和高质量阴影叠加时容易出现掉帧。鼠标按住拖动主窗口时，也会因为实时渲染阴影和背景效果而显得卡顿。

原因：

主窗口使用透明 WPF 窗口承载圆角和阴影，主面板又叠加 `DropShadowEffect` 与背景视觉效果。原有 XamlFlair 横向展开动画会让整块大面板参与动画渲染，拖动窗口时同样需要持续重绘这些昂贵效果。

优化：

- 主面板启用 `BitmapCache`，减少重复渲染压力。
- 主窗口动画由横向展开改为轻量的淡入和轻微缩放。
- 关闭动画改为短淡出，避免拖泥带水。
- 阴影渲染从 `Quality` 调整为 `Performance`。
- 拖动窗口和播放主窗口动画时临时暂停阴影，结束后恢复，提升拖动跟手感。

相关文件：

- `MainWindow.xaml`
- `MainWindow.xaml.cs`

验证：

- Release 构建通过，0 个错误。
